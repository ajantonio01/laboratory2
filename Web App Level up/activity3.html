Activity #3


How would you design an interface for a 1000 floor elevator?

  • My ideal design interface for a 1000 floor elevator would be having a touch screen monitor inside the elevator. This monitor has a search bar that can accept input from the user. The purpose of this interface is to let the user type in the search bar by typing the number of the floor they want to be, after the user finishes choosing a floor, the elevator will now accommodate the user’s request by transporting the user to the chosen floor. For instance, a passenger wanted to go to 984 th floor, he/she then typed 984 to the search bar, after typing the number, the 984 th floor button will show to the screen, the user will now tap that button and alerted a message saying “Do you want to go to 984 th floor? Yes or No?”, if the user tapped the button “Yes”, the elevator will accommodate the user’s request by going to the 984 th floor. However, if the user tapped the button “No”, the message will simply disappear, and the user can freely interact with the monitor again. I designed this interface to be easy-to-use for the user, so user can still casually ride the elevator no matter the number of floors it has. In addition, the input can only accept integers/numbers from 1-1000, the user will receive an error message if they try to input other than the fixed numbers.
